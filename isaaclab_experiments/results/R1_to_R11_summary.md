# IsaacLab R1 到 R11 简要复盘

这份记录是为了说明：最终成果不是直接从 Booster Gym 开始的，中间有一段 IsaacLab 探索。下面按路线而不是严格 run id 复盘。

| 轮次 | 方向 | 主要目标 | 结论 |
|---|---|---|---|
| R1 | velocity flat T1 | 让 T1 在 IsaacLab 中完成基本速度跟踪 | 任务能注册，但资产、关节、脚接触和奖励还需要大量修正 |
| R2 | T1 asset / joint layout | 明确 T1 23DOF/27DOF、左右腿关节顺序和默认姿态 | 为后面的镜像映射和动作转换打基础 |
| R3 | human-ref | 尝试用人体参考动作约束 T1 步态 | 形态约束有帮助，但受动作数据和机器人 DOF 差异影响较大 |
| R4 | symclock | 用相位/时钟约束左右脚周期 | 能形成节律思路，但稳定性和速度扩展仍不理想 |
| R5 | recurrent / mirror | 尝试 recurrent 策略和镜像约束 | 工程成本上升，训练效果没有明显超过简单课程 |
| R6 | biomech curriculum | 用更像人体的阶段课程推进速度 | 阶段思想保留下来，但 IsaacLab 路线整体收敛慢 |
| R7 | quiet upper | 压低上身和手臂多余动作 | 上半身更安静，但对下肢速度跟踪提升有限 |
| R8 | full symmetry | 强化全身左右对称 | 对称性改善与速度/稳定性之间需要重新权衡 |
| R9 | arm envelope explore | 给手臂动作限制活动包络 | 对观感有帮助，但不是当时最大瓶颈 |
| R10 | fullspeed tasks | 直接推动更高速度 | 容易出现追速但姿态/接触质量差的问题 |
| R11 | fullspeed momentum arm | 尝试利用手臂动量辅助高速 | 思路有价值，但没有在 IsaacLab 中形成满意最终成果 |

## 关键经验

1. 早期最大问题不是单个奖励项，而是工程闭环太慢：注册任务、Windows 兼容、资产路径、数据转换、播放和评分都要同时处理。
2. 对称性确实重要，但不能一上来压得太重，否则容易牺牲速度跟踪和自然摆动。
3. 最后成功的 Booster Gym S0-S8，其实吸收了 IsaacLab 阶段的经验：先拆阶段，再逐步加命令空间；先把能跑稳，再处理对称和脚滑。
